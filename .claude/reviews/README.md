# Reviews — Policy & Index

Log significant reviews here (design critiques, architecture reviews, security audits) when they produce decisions worth remembering beyond the PR/conversation they happened in. Not every code review needs an entry — this is for structured, multi-finding review passes.

**Policy:** after a review, ask (per `CLAUDE.md` §8 Continuous Learning): should this become a rule, checklist, pattern, or documentation update? If yes, create it and link it from the review log entry — don't leave the finding only in this log.
