# Prompt: Documentation Writing

```
Write documentation for [topic].
Before writing: search docs/ and .claude/ for an existing file covering this — extend it, don't duplicate.
Structure: Why (motivation/constraint) → What (the fact/decision) → How (how to apply it) → Tradeoffs/Limitations → cross-links to related docs.
Decide placement: docs/ (product/architecture fact, PO-owned) vs .claude/ (AI operating knowledge).
Do not restate a fact that already has a canonical home — link instead.
If this is a docs/ change to a product/architecture fact, present it for user confirmation before treating it as final.
```
